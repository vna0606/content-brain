### 1.0 Tone of Voice — самое важное ⭐

**Проблема сейчас:** посты загружены в Turso, но Claude ничего не знает о том КАК ты пишешь. В `strategy.md` пусто. В промпте только общие требования: "200-500 слов, живой стиль". Результат будет безликим.

**Что нужно сделать — два шага:**

#### Шаг А: Извлечь стиль из существующих постов (скрипт, ~30 мин работы)

- [ ] Написать скрипт `03-bot/extract_tone.py`:
  - Читает все посты из `cb_social_posts` (source_type='telegram')
  - Отправляет их Claude с промптом: "Проанализируй эти посты. Выдели: длину предложений, структуру абзацев, как автор начинает посты, как заканчивает, какие обороты повторяются, что делает эти посты узнаваемыми, какой ритм у текста."
  - Сохраняет анализ в `03-bot/tone_analysis.md`
- [ ] Просмотреть результат, скорректировать вручную
- [ ] Результат становится базой для `strategy.md`

#### Шаг Б: Заполнить strategy.md

Структура файла должна быть такой:

```
## Кто я и о чём канал
[2-3 предложения: позиционирование]

## Темы которые пишу
[список]

## Темы которые НЕ пишу
[список]

## Как выглядит хороший пост (структура)
[описание структуры на основе анализа из шага А]

## Мой тон голоса
[характеристики из шага А: ритм, обороты, что делать и чего не делать]

## Примеры постов (5 штук — лучшие или понравившиеся чужие)
[вставить тексты целиком]
```

> Примеры постов — самое важное. Если твои посты не идеальны — 
> можно вставить чужие посты которые нравятся по стилю, 
> и Claude будет ориентироваться на них.

- [ ] Запустить `extract_tone.py` → получить анализ
- [ ] Заполнить `strategy.md` на основе анализа + своих ощущений
- [ ] Вставить 5 примеров хороших постов (своих или чужих для стиля)

### 1.4 Умные ссылки на старые посты (опционально, не всегда)

**Что хотим:** не постоянные ссылки, а умная подсказка.

Два сценария:
1. **Пользователь упомянул** ("там я писал об X") → Claude ищет нужный пост и подставляет ссылку
2. **Claude сам предлагает** → отдельная кнопка "Найти связанные посты", не автоматически

**Реализация:**

- [ ] Функция `find_related_posts(query, top_k=3)` в `post_writer.py`:
  - Embed query → cosine similarity с `cb_social_vectors`
  - Возвращает список `(content_snippet, url)` с score > 0.75
- [ ] Кнопка "Связанные посты" в меню черновика — запускает поиск и присылает список
  - Пользователь решает: вставлять ссылку или нет
- [ ] Если в тексте идеи пользователь написал "я писал об X" — автоматический поиск перед генерацией

> Не делать автопоиск при каждой генерации — только по запросу или по триггеру в тексте.

## Этап 3 — Длинные видео: транскрипция и текстовый анализ

**Цель:** закинул видео в Google Drive → сервер транскрибирует → бот присылает анализ.

**Статус:** Не начат  
**Файлы:** новый `04-video/`

### Архитектура

```
[Мак] снял видео → Google Drive /content-brain/inbox/
         ↓
[Сервер] polling Google Drive (раз в 5 мин)
       → скачать видео во /tmp/cb-video/
       → ffmpeg: извлечь аудио (wav, 16kHz mono)
       → удалить видеофайл (место на сервере)
       → Groq Whisper API: транскрибация (~1 мин на 10 мин видео)
       → Claude анализ транскрипта:
           - что убрать (куски с примерным местом в тексте)
           - как сократить без потери смысла
           - 3 варианта названия
           - описание для YouTube
           - теги (10-15 штук)
       → сохранить в cb_video_tasks
       → бот присылает на одобрение
         ↓
[Мак] после одобрения — ручной монтаж
```

### 3.1 Новые таблицы в Turso

```sql
CREATE TABLE cb_video_tasks (
    id INTEGER PRIMARY KEY,
    gdrive_file_id TEXT,
    original_filename TEXT,
    transcript TEXT,
    analysis TEXT,       -- JSON: {cuts, title_options, description, tags}
    status TEXT DEFAULT 'analyzing',  -- analyzing|ready|approved|done
    created_at TEXT
);
```

### 3.2 Сервисный скрипт `04-video/video_processor.py`

- [ ] Google Drive API: список файлов в `/content-brain/inbox/`
  - Нужны credentials — **проверить что уже есть в экосистеме** (есть Google Drive MCP)
- [ ] Скачивание видео → `/tmp/cb-video/`
- [ ] `ffmpeg -i input.mp4 -vn -ar 16000 -ac 1 output.wav`
- [ ] Groq Whisper API: транскрипция
- [ ] Удалить локальное видео + аудио после транскрипции
- [ ] Claude с транскриптом → структурированный анализ (JSON)
- [ ] Запись в `cb_video_tasks`
- [ ] Уведомление в бот через TELEGRAM_USER_ID
- [ ] Cron: каждые 5 минут (или по расписанию)

### 3.3 Handler в боте `/videos`

- [ ] Показ анализа: что убрать + 3 варианта названия + описание + теги
